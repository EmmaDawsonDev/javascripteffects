Deep Dive into the DOM

The DOM

The DOM (short for Document Object Model) is a programming interface created from the HTML code when a browser loads. It is the standard way of accessing documents as defined by the World Wide Web Consortium (W3C). There are three parts to the DOM: Core, XML and HTML. They are standard models for different document types. Core is standards for all document types, XML is standard for XML documents and HTML is standard for HTML. Here we will focus mainly on the HTML DOM.


The DOM is a representation of your webpage and can be thought of like a family tree, where different HTML elements branch from one another in parent-child or sibling relationships. Each element in the DOM is called an object. The DOM also defines properties, methods and events for all HTML elements.


Everything in the DOM is also called a node. There are several types of nodes: the entire document is a document node, all HTML elements are element nodes and text inside the HTML are text nodes. Each node can have only one parent (except the root which has no parent) but any number of children or siblings. Siblings are nodes which share the same parent node.




The DOM can be used by Javascript (and other programming languages) to dynamically manipulate HTML nodes and change the look and behaviour of a page without having to reload the page every time. Javascript is able to access all nodes in the DOM and is able to add, remove and change the html elements themselves, or the properties applied to them. It can be used to modify HTML attributes and CSS styling. It can also react to HTML events.


There are several ways to access nodes in the DOM in order to manipulate them. Elements can be reached by id, tag name, class name, CSS selectors or HTML object collections. Dot notation is used to tell Javascript to traverse the DOM tree. Here is an example:


document.getElementsByTagName(‘p’);


This will search the document for all elements with the tag and return an array which can then be manipulated with Javascript.


Javascript is also able to manipulate HTML through HTML events. Such events can include button clicks, keystrokes, mouse movement, page loading, and modification of input fields. Javascript can add ‘Event Listeners’ to these events so that when an event is triggered in the HTML the Javascript can then execute a piece of code which changes the HTML. These changes can include changes to CSS styles, addition of innerHTML to tags, addition or removal of HTML tags or changes to HTML attributes. It is even possible to create animations by using Javascript to manipulate the HTML DOM.

Local Storage

Local storage is a part of the DOM which allows information to be stored securely and directly in the client’s browser in the form of key-value pairs without the need for information being sent back and forth to a server. The information stored in local storage persists between browser sessions and does not expire. This means a page can remember what was previously added and is useful for both website customisation and performance reasons. Each site has a separate storage area.

To access the local storage we use dot notation as follows:

window.localStorage;

The local storage is read only and can only accept information stored in strings. Therefore, information stored in Javascript objects needs to be converted to a string using JSON and parsed back from string to object when being retrieved from local storage.

There are four methods we can use to manipulate information in the local storage:
.setItem() - adds information to the local storage
.getItem() - retrieves information from the local storage
.removeItem() - removes an item from the local storage
.clear() - removes all items stored in the local storage
